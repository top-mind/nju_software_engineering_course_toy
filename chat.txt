GitHub Copilot: 需求分析是软件开发过程中非常重要的一步，通常包括以下内容：

1. **项目概述**：
   - 项目背景
   - 项目目标
   - 项目范围

2. **功能需求**：
   - 用户需求
   - 系统功能描述
   - 用例图和用例描述

3. **非功能需求**：
   - 性能需求（响应时间、吞吐量等）
   - 安全需求
   - 可用性需求
   - 兼容性需求
   - 可维护性需求

4. **用户界面需求**：
   - 界面原型
   - 界面设计规范

5. **数据需求**：
   - 数据模型
   - 数据存储需求
   - 数据流图

6. **系统接口需求**：
   - 外部系统接口
   - API 需求

7. **业务流程**：
   - 业务流程图
   - 业务规则

8. **约束条件**：
   - 技术约束
   - 法律和法规约束
   - 项目时间表和预算

9. **风险分析**：
   - 潜在风险
   - 风险应对策略

10. **验收标准**：
    - 验收测试计划
    - 验收标准和方法

这些内容可以帮助团队明确项目的需求，确保开发过程中的各个环节都有明确的指导和依据。

- 权限管理：不同用户角色拥有不同的权限
